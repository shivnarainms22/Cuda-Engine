# GEMM Rung 1 (Foundation + De-risk) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a small `gemm` eval suite (square GEMM + a fused-epilogue probe) and the CLI wiring to run it, so a single credited Colab A100 spike can answer honestly: can the engine generate correct tiled GEMM, and what fraction of cuBLAS does it reach?

**Architecture:** Reuse the existing fixture format + eval runner unchanged. All fixtures are **square** (M=K=N) so the current input generator produces matmul-compatible shapes with no correctness-machinery change. GEMM-specific guidance lives in the fixture `prompt.txt` (not the codegen stage), so Rung 1 makes **zero engine code changes** — it's a pure experiment.

**Tech Stack:** Python 3.11, pytest, torch (CPU for the fixture-sanity unit test). No hardware for unit tests; the spike is the only credit-gated step.

**Spec:** `docs/superpowers/specs/2026-06-29-v2.0-gemm-rung1-design.md`.

## Key facts
- Fixture format (per kernel dir): `prompt.txt`, `reference.py` (`def reference(...)`), `shapes.yaml` (list of shapes), `notes.md`. See `evals/internal/*` and `tests/unit/test_internal_eval_suite.py`.
- Input generator (`stages/correctness.py`): fp16 inputs are `(arange%17-8)/8 + arg_index` (range ~[-1,1]); fp32 inputs are raw `arange + arg_index`. Symbolic dims bind positionally per-arg → **square shapes give compatible matmul inputs**.
- Suite-name → dir map: `_resolve_suite_root` in `src/cuda_engine/cli.py` (`internal`, `kernelbench`). The runner handles any suite dir.
- Existing names to avoid colliding with: the 42 internal + 12 kernelbench kernels.

## Chunk 1: GEMM fixtures + suite test

### Task 1: Create the `evals/gemm/` fixtures
**Files:** Create `evals/gemm/<name>/{prompt.txt,reference.py,shapes.yaml,notes.md}` for each fixture below; Test `tests/unit/test_gemm_suite.py`.

Fixtures (all **square**; fp16 inputs land in ~[-1,1] so matmul outputs stay finite):
- `matmul_fp16` — `def reference(a, b):\n    import torch\n    return a @ b`. Prompt: "fp16 matrix multiply C = A @ B, A and B square N×N, fp16 in/out; use a shared-memory tiled approach for coalesced loads." shapes.yaml: `- [256, 256]` / `- [512, 512]` / `- [1024, 1024]`.
- `matmul_fp32` — same reference, fp32. Prompt notes CUDA-core tiling (no tensor cores). Same square shapes.
- `matmul_bias_gelu_fp16` — `def reference(a, b, bias):\n    import torch\n    return torch.nn.functional.gelu(a @ b + bias)`. bias shape `[N]` broadcasts over rows. Prompt: fused tiled GEMM with bias add + tanh-GELU epilogue in one kernel. shapes: square; the suite test must confirm bias broadcasts (a@b is N×N, bias is N → broadcast over rows).

- [ ] **Step 1: Write the failing suite test** `tests/unit/test_gemm_suite.py`:
  - `discover_kernels(evals/gemm)` finds exactly the fixture names; each has the 4 required files and ≥3 shapes; references import and are callable.
  - **No name overlap** with the internal (42) or kernelbench (12) suites.
  - References produce **finite** outputs under the real input generator (mirror the finite-check used when the v1.1 kernels were added: build fp16 inputs `(arange%17-8)/8 + idx`, fp32 `arange + idx`, square shapes, assert `torch.isfinite(out).all()`). For `matmul_bias_gelu_fp16`, generate `a,b` as N×N and `bias` as N.
- [ ] **Step 2: Run, expect FAIL** (suite dir / fixtures absent).
- [ ] **Step 3: Create the fixture files** as specified.
- [ ] **Step 4: Run, expect PASS**; full `pytest tests/unit --ignore=tests/unit/test_web_demo.py` still **0 failed**.
- [ ] **Step 5: Commit** `feat(evals): gemm suite — square matmul + fused-epilogue probe fixtures`.
