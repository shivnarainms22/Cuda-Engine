# v1.1 "Sharpen" — Evidence

Branch `v1.1/sharpen` (off `main` at v1.0.0). Increment #1 of the post-v1.0
sequence. Spec: `docs/superpowers/specs/2026-06-28-v1.1-sharpen-design.md`.

## What shipped
1. **Multi-provider LLM layer** — capability-aware, native-first. `LLMRouter`
   dispatches on namespaced model ids; native `Anthropic`/`OpenAI`/`Gemini`
   adapters + a generic `OpenAICompatibleClient` for any OpenAI-API endpoint;
   per-stage routing via `SynthesisConfig.stage_models`; `cuda-engine eval
   --model-id` to run the suite on any provider. Claude stays the default with
   prompt caching + tool use; weaker providers degrade gracefully and the run
   records it.
2. **Bound-aware Stage-4 perf repair** — `parse_ncu_csv` now extracts the
   bottleneck signals (memory/dram/compute %, waves/SM, SOL verdict) it had been
   discarding; `classify_bound` labels latency/bandwidth/compute; the repair
   prompt leads with targeted guidance instead of defaulting to "add more ILP".
3. **+12 in-scope internal eval kernels** — internal suite 30 → 42.

Deferred to v1.2 (spec §2.5): automated cross-provider comparison report.
